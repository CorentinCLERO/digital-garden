# Mise en place d’un VPN personnel avec Tailscale

## 1. Pourquoi utiliser un VPN pour son serveur ?

- **Sécurité** : Accès privé à son serveur, sans exposer de ports sur Internet.
- **Simplicité** : Connexion à son serveur comme si on était sur le même réseau local, où que l’on soit.
- **Flexibilité** : Possibilité d’accéder à d’autres appareils du réseau local via le VPN.

## 2. Présentation de Tailscale

- **Tailscale** est un VPN moderne basé sur WireGuard, open source, gratuit pour un usage personnel.
- Il crée un réseau privé virtuel (“tailnet”) entre tous les appareils connectés avec le même compte.
- Il gère automatiquement le chiffrement, la découverte des pairs, et le contournement des NAT/box.
- [Site officiel Tailscale](https://tailscale.com/)

## 3. Installation de Tailscale sur le serveur (Ubuntu)

- Télécharger et installer Tailscale :
    ```bash
    curl -fsSL https://tailscale.com/install.sh | sudo bash
    ```
- Démarrer Tailscale et authentifier l’appareil :
    ```bash
    sudo tailscale up
    ```
    - Un lien s’affiche : l’ouvrir dans un navigateur et se connecter avec son compte (GitHub, Google, etc.).
    - Si le serveur n’a pas d’interface graphique, générer une clé d’authentification sur [l’interface web Tailscale](https://login.tailscale.com/admin/settings/keys) et lancer :
        ```bash
        sudo tailscale up --authkey tskey-xxxxxxxxxxxxxxxx
        ```

## 4. Installation de Tailscale sur les autres appareils

- Installer Tailscale sur son PC principal (Linux, Windows, Mac, Android, iOS).
- Se connecter avec le même compte que sur le serveur.
- Les appareils rejoignent automatiquement le même réseau privé (tailnet).

## 5. Vérification du fonctionnement

- Sur chaque appareil, vérifier la liste des machines connectées :
    ```bash
    tailscale status
    ```
- Chaque machine possède une adresse IP privée Tailscale (ex : `100.95.12.35`).
- Tester la connexion SSH via Tailscale :
    ```bash
    ssh utilisateur@100.95.12.35
    ```
    ou, si MagicDNS est activé :
    ```bash
    ssh utilisateur@nom-machine
    ```

## 6. Activer MagicDNS (résolution de noms simplifiée)

- Aller sur [l’interface DNS Tailscale](https://login.tailscale.com/admin/dns).
- Activer **MagicDNS** pour pouvoir utiliser les noms de machines au lieu des adresses IP.
- Exemple de connexion :
    ```bash
    ssh utilisateur@nom-machine
    ```
    ou
    ```bash
    ssh utilisateur@nom-machine.tailXXXXX.ts.net
    ```

## 7. Contrôler l’accès avec des tags et des ACLs

- Pour restreindre l’accès SSH à certaines machines, utiliser les **tags** et les **ACLs** dans la configuration Tailscale.
- Exemple :
    1. Ajouter un tag à la machine serveur :
        ```json
        "tagOwners": {
          "tag:server": ["CorentinCLERO@github"]
        }
        ```
    2. Lancer Tailscale sur le serveur avec le tag :
        ```bash
        sudo tailscale up --advertise-tags=tag:server
        ```
    3. Restreindre l’accès SSH dans la section `"grants"` :
        ```json
        "grants": [
          {
            "src": ["CorentinCLERO@github"],
            "dst": ["tag:server"],
            "ip": ["22"]
          }
        ]
        ```
    - Modifier la configuration ACL sur [l’interface d’administration Tailscale](https://login.tailscale.com/admin/acls).

- [Documentation officielle sur les ACLs](https://tailscale.com/kb/1018/acls/)

## 8. (Optionnel) Utiliser Tailscale SSH

- Tailscale SSH permet de se connecter en SSH sans gérer de clés SSH manuellement.
- Activer Tailscale SSH sur le serveur :
    ```bash
    sudo tailscale up --ssh
    ```
- Contrôler l’accès via la section `"ssh"` dans la configuration ACL.
- [Documentation Tailscale SSH](https://tailscale.com/kb/1193/tailscale-ssh/)

## 9. (Optionnel) Accéder à tout le réseau local (subnet router)

- Pour accéder à tous les appareils du réseau local via Tailscale :
    ```bash
    sudo tailscale up --advertise-routes=192.168.1.0/24
    ```
    (adapter le sous-réseau à celui de la box)
- Activer le routage sur l’interface web Tailscale (Machines > Enable subnet routes).
- [Documentation Subnet Router](https://tailscale.com/kb/1019/subnets/)

---

**Avec cette configuration, le serveur est accessible de façon sécurisée et privée, où que l’on soit, sans ouvrir de ports sur la box et sans exposer de services sur Internet.**
