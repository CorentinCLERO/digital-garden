# Mise en production d’un serveur web sécurisé sur Ubuntu

## 1. Configuration du pare-feu (UFW)

### Installation et configuration de base

```bash
sudo apt install ufw
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw enable
```

- **ufw** : Uncomplicated Firewall, outil simple pour gérer le pare-feu sous Ubuntu.
- **default deny incoming** : Bloque toutes les connexions entrantes par défaut.
- **default allow outgoing** : Autorise toutes les connexions sortantes.
- **enable** : Active le pare-feu avec les règles définies.

> **Note** : Si tu utilises Tailscale SSH ou un autre VPN sécurisé, il n’est pas nécessaire d’ouvrir le port SSH sur le pare-feu.

---

## 2. Installation et configuration de Nginx

### Installation

```bash
sudo apt install nginx
```

- Installe le serveur web Nginx et ses dépendances.

### Ouverture des ports HTTP/HTTPS dans le pare-feu

```bash
sudo ufw allow 'Nginx Full'
```

- Ouvre les ports 80 (HTTP) et 443 (HTTPS) pour permettre l’accès web.

### Vérification de l’état de Nginx

```bash
sudo ss -tulnp | grep nginx
```

- Vérifie que Nginx écoute bien sur le port 80 (HTTP).

---

## 3. Redirection de ports sur la box Internet

- Accéder à l’interface d’administration de la box (ex : http://192.168.1.1).
- Aller dans la section “NAT/PAT” ou “Redirections de ports”.
- Ajouter deux règles :
    - **HTTP** : Port externe 80 → IP locale du serveur :80
    - **HTTPS** : Port externe 443 → IP locale du serveur :443
- (Optionnel) Ne pas ouvrir le port 22 (SSH) si tu utilises Tailscale ou un autre VPN pour l’accès distant.

## 4. Obtenir et configurer un nom de domaine et sa zone DNS

### Achat du nom de domaine

1. Choisir un registrar (ex : OVH, Gandi, Namecheap…).
2. Rechercher le nom de domaine souhaité et vérifier sa disponibilité.
3. Procéder à l’achat et attendre la confirmation.

### Activation de la zone DNS (exemple OVH)

1. Se connecter à l’espace client OVH : [https://www.ovh.com/manager/](https://www.ovh.com/manager/)
2. Aller dans “Web Cloud” > “Domaines” > [ton-domaine].
3. Si la zone DNS n’est pas activée, cliquer sur “Activer la zone DNS” et valider.

### Gestion de la zone DNS

- Accéder à “Zone DNS” pour ajouter/modifier/supprimer des enregistrements.

### Configuration des enregistrements DNS

#### Enregistrement A (domaine principal et www)

| Type | Sous-domaine | Cible (IP)      | TTL  |
|------|--------------|-----------------|------|
| A    | @            | 123.45.67.89    | 3600 |
| A    | www          | 123.45.67.89    | 3600 |

- **@** : domaine principal (ex : sleipnirproject.fr)
- **www** : sous-domaine www.sleipnirproject.fr
- **Cible** : IP publique de ton serveur (trouvée avec `curl ifconfig.me` ou sur [monip.org](https://www.monip.org/))

#### Enregistrement pour un sous-domaine (optionnel)

| Type | Sous-domaine | Cible (IP)      | TTL  |
|------|--------------|-----------------|------|
| A    | blog         | 123.45.67.89    | 3600 |

### Propagation DNS

- Attendre la propagation (quelques minutes à 24h).
- Tester avec :
    ```bash
    nslookup sleipnirproject.fr
    ```

### (Optionnel) DNS dynamique

- Si l’IP publique change régulièrement, utiliser un service de DNS dynamique (DynHost OVH, DuckDNS, etc.) pour mettre à jour automatiquement l’enregistrement A.

## 5. Sécuriser son site avec HTTPS (Let’s Encrypt, Certbot, Nginx)

### Pourquoi HTTPS ?

- Chiffre les échanges entre visiteurs et serveur.
- Rassure les utilisateurs (cadenas dans le navigateur).
- Améliore le référencement Google.
- Let’s Encrypt fournit des certificats SSL/TLS gratuits et automatisés.

### Installer Certbot et le plugin Nginx

```bash
sudo apt update
sudo apt install certbot python3-certbot-nginx
```

- **certbot** : Outil pour demander, installer et renouveler les certificats SSL/TLS.
- **python3-certbot-nginx** : Plugin pour configurer automatiquement Nginx.

### Générer et installer le certificat avec Certbot

#### Pour des domaines/sous-domaines précis

```bash
sudo certbot --nginx -d sleipnir.ovh -d www.sleipnir.ovh -d todoapp.sleipnir.ovh
```

- Liste tous les domaines/sous-domaines à sécuriser avec `-d`.

#### Pour un certificat wildcard (tous sous-domaines)

```bash
sudo certbot --nginx -d sleipnir.ovh -d '*.sleipnir.ovh' --manual --preferred-challenges dns
```

- Certificat wildcard nécessite une validation DNS (ajout d’un enregistrement TXT).

### Ce qui se passe techniquement

1. Certbot contacte Let’s Encrypt pour demander un certificat.
2. Let’s Encrypt vérifie le contrôle du domaine (challenge HTTP ou DNS).
3. Si OK, Let’s Encrypt délivre le certificat.
4. Certbot installe le certificat et modifie la config Nginx.
5. Certbot programme un renouvellement automatique.

#### Vérification

- Ouvrir le site dans un navigateur : le cadenas HTTPS doit apparaître.
- HTTP doit rediriger automatiquement vers HTTPS.

#### Renouvellement automatique

- Certbot installe un cron job ou un timer systemd pour renouveler automatiquement le certificat.
- Tester le renouvellement :
    ```bash
    sudo certbot renew --dry-run
    ```
- Pour les wildcards, le renouvellement nécessite de refaire la validation DNS (sauf automatisation).

### Bonnes pratiques

- Toujours inclure tous les domaines/sous-domaines à sécuriser lors de la génération/renouvellement.
- Pour de nombreux sous-domaines, privilégier le certificat wildcard (prévoir la gestion DNS).
- Vérifier régulièrement que le renouvellement automatique fonctionne.

---

**Avec ces étapes, ton serveur est sécurisé, accessible via un nom de domaine, et prêt à héberger des applications web en production.**
