# Documentation : Monitoring avec Netdata et Uptime Kuma

## 1. Netdata

### 1.1. Installation

```bash
sudo apt update
sudo apt install netdata
```

### 1.2. Configuration

- Par défaut, Netdata écoute uniquement sur `localhost`.
- Pour le rendre accessible sur le réseau local ou via Tailscale, édite `/etc/netdata/netdata.conf` :

```ini
[global]
  bind socket to IP = 0.0.0.0
```
ou, pour n’écouter que sur l’IP Tailscale (plus sécurisé) :

```ini
[global]
  bind socket to IP = [IP_TAILSCALE_DU_SERVEUR]
```

- Redémarre Netdata :

```bash
sudo systemctl restart netdata
```

- Ouvre le port 19999 dans le firewall **si besoin** (inutile si accès uniquement via Tailscale) :

```bash
sudo ufw allow 19999/tcp
```

### 1.3. Accès sécurisé (Tailscale)

- Installe et connecte Tailscale sur le serveur et le client.
- Accède à Netdata via l’IP Tailscale du serveur :

```
http://[IP_TAILSCALE_DU_SERVEUR]:19999
```

- Pour plus de sécurité, supprime la règle UFW sur le port 19999 si tu utilises uniquement Tailscale :

```bash
sudo ufw delete allow 19999/tcp
```

## 2. Uptime Kuma

### 2.1. Installation avec Docker

Crée un dossier pour Uptime Kuma et un fichier `docker-compose.yml` :

```yaml
version: "3.3"
services:
  uptime-kuma:
    image: louislam/uptime-kuma:latest
    volumes:
      - uptime-kuma:/app/data
    network_mode: host
    restart: always
volumes:
  uptime-kuma:
```

**Remarque :**  
- Avec `network_mode: host`, ne mets pas de section `ports:`.
- Ce mode permet à Uptime Kuma de monitorer tous les services du host (localhost, IP locale, Tailscale…).

Lance Uptime Kuma :

```bash
sudo docker-compose up -d
```

Accède à l’interface web :

```
http://localhost:3001
```
ou
```
http://[IP_TAILSCALE_DU_SERVEUR]:3001
```

### 2.2. Configuration réseau pour le monitoring local

- Grâce au mode `host`, tu peux monitorer :
  - Site Internet (mon site racine) : `https://sleipnir.ovh`
  - Netdata : `http://localhost:19999`
  - SSH : `localhost:22`
  - Tout autre service local ou sur une IP du serveur

### 2.3. Utilisation de base

1. **Première connexion** : crée un compte admin.
2. **Ajouter un moniteur** :
   - HTTP(s) : pour un site web ou une API
   - TCP Port : pour SSH, Netdata, etc.
   - Ping : pour vérifier la disponibilité d’une machine
3. **Configurer les notifications** (email, Telegram, etc.) si besoin.
4. **Créer une page de statut publique** (optionnel).

### 2.4. Accès sécurisé (Tailscale)

- Si tu utilises Tailscale, accède à Uptime Kuma via l’IP Tailscale du serveur.
- Pas besoin d’ouvrir le port 3001 dans le firewall si tu passes uniquement par Tailscale.
